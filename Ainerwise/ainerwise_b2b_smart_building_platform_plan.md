# AinerWise B2B 智能建筑/能源系统集成平台规划

> 定位：面向塞尔维亚起步、后续扩展到波兰、新西兰、澳洲等区域的 B2B 智能建筑解决方案平台。核心不是“卖设备商城”，而是“解决方案 + 供应链 + 本地服务 + 长期运维”的系统集成公司数字化平台。

---

## 1. 一句话定位

AinerWise 是一个面向企业客户、酒店、公寓、别墅、商铺、小商业楼、光伏/储能/智能化改造项目的 B2B 智能建筑解决方案平台。

平台提供：

- KNX / Home Assistant / MQTT / CCTV / Access Control / HVAC / Solar Monitoring / Energy Management 等方案服务；
- 国内与本地供应商产品库；
- 项目需求发布与 AI 初步方案生成；
- 平台官方产品销售；
- 供应商入驻；
- 本地施工、调试、维护、保修、长期运维服务；
- Telegram 自动通知与项目群协作；
- 多区域、多语言、多币种扩展能力。

核心商业逻辑：

> 客户买的不是设备，而是“可落地、可维护、可长期服务的智能建筑系统”。

## 2. 是否值得做？我的判断

建议做，但不要一开始做成“大而全电商平台”。

更适合你的第一版是：

> “解决方案官网 + B2B 需求收集 + 供应商产品库 + 后台项目管理 + Telegram 通知 + AI 方案助手”

而不是一开始就做成完整阿里巴巴式 marketplace。

原因：

1. 你现在还在准备 KNX / 智能化落地阶段，真实项目经验还在积累；
2. B2B 客户不会像 C 端一样直接下单买复杂系统；
3. 智能建筑项目强依赖现场条件、施工、售后、保修；
4. 设备只是方案的一部分，真正利润来自集成、服务、维护和长期关系；
5. 你需要先用网站建立可信度、收集需求、沉淀供应链，而不是马上追求交易闭环。

最优策略：

- V1：品牌官网 + 解决方案展示 + 需求表单 + AI 初步评估 + 后台管理；
- V2：供应商入驻 + 产品库 + 项目匹配 + Telegram 通知；
- V3：报价、合同、订单、服务包、运维、质保、区域扩展；
- V4：AI 自动方案 + 采购清单 + 成本估算 + 运维预测。

## 4. 平台核心模式

### 4.1 不建议做纯商城

智能建筑/KNX/HVAC/光伏监控这类业务，不适合像普通电商一样：

- 客户看商品；
- 加购物车；
- 直接付款；
- 快递发货。

因为客户真正需要的是：

- 现场评估；
- 设备兼容性；
- 安装调试；
- 售后维护；
- 质保；
- 系统扩展；
- 长期稳定。

所以应该做成：

> 商品可展示，但购买动作默认进入“询价 / 项目需求 / 服务确认”流程。

---

### 4.2 推荐的商业结构

平台分三类业务：

#### A. 官方解决方案服务

由 AinerWise 官方提供：

- 智能家居基础改造；
- 酒店智能化；
- 公寓/别墅智能化；
- CCTV / 门禁 / 网络；
- KNX 调试；
- Home Assistant 集成；
- Solar / Energy Dashboard；
- 设备升级维护；
- 远程运维；
- 年度服务包。

这是你前期最重要的收入来源。

#### B. 官方产品销售

你作为管理员可以发布 Official 产品：

- 自己代理的国内设备；
- 自己选品的高性价比设备；
- 已测试兼容的套装；
- 官方推荐配件；
- 方案包。

这些产品可以带服务费、安装费、质保服务包。

#### C. 供应商入驻

国内或本地供应商可以加入：

- 上传产品；
- 设置 B2B 价格；
- 设置 MOQ；
- 设置交期；
- 设置质保条款；
- 设置是否支持本地服务；
- 设置备用件比例；
- 接收询盘和项目机会。

但前期建议供应商产品必须由管理员审核，不要完全开放。

建议报价结构：

```text
Device Cost
+ Platform Margin
+ Integration Service Fee
+ Installation / Commissioning Fee
+ Warranty / Support Package
+ Spare Parts Reserve
+ Logistics / Tax
= Total Project Price
```

前台显示方式：

- 对普通客户：显示打包报价，不要把所有成本拆太细；
- 对企业客户：可以显示设备、安装、服务包、维护费；
- 对后台：所有成本必须清楚拆分。

## 9. 供应商备用件与质保策略

你的想法是对的：如果卖 10 台，实际可能要准备 12-13 台，其中 2-3 台作为替换/备用件。

但不要简单写死为所有供应商都必须 30% 备用件，因为不同设备故障率不同。

建议做成后台可配置：

### 9.1 备用件策略字段

- spare_policy_type：none / percentage / fixed_quantity / project_based
- spare_percentage：例如 5%、10%、15%、20%
- spare_owner：vendor / platform / client
- spare_location：China warehouse / Serbia local / client site / installer
- spare_valid_years：3 / 5 / 8 / 10
- replacement_response_days：7 / 14 / 30

### 9.2 推荐默认策略

#### 普通低价设备

- 备用件比例：5%-10%；
- 服务期：1-3 年；
- 适合预算项目。

#### 核心控制设备

- 备用件比例：10%-20%；
- 服务期：3-8 年；
- 建议放本地备用。

#### 酒店/商业项目

- 按项目配置备用件；
- 关键设备必须本地备件；
- 服务合同单独报价。

## 10. AI 方案助手设计

你可以用 LangGraph 做后台 AI workflow，但不要让 AI 直接替你承诺最终方案。

AI 的角色应该是：

> 初步分析助手 + 信息收集助手 + 推荐草案生成器 + 报价辅助器。

最终方案由你或工程师确认。

### 10.1 AI 工作流

客户提交需求后：

1. 信息完整性检查；
2. 缺失信息追问；
3. 项目类型判断；
4. 预算等级判断；
5. 推荐系统架构；
6. 推荐产品类别；
7. 匹配供应商；
8. 生成初步方案草案；
9. 生成采购清单草案；
10. 生成风险提醒；
11. 通知管理员审核。

### 10.2 AI 输入

- 项目类型；
- 面积；
- 房间数；
- 预算；
- 国家/城市；
- 现场照片；
- 图纸；
- 客户目标；
- 现有设备；
- 语言；
- 产品库；
- 供应商能力；
- 服务范围。

### 10.3 AI 输出

- 项目摘要；

### 11.2 群组策略

建议三类群：

#### A. Admin Internal Group

只有你和内部团队。

收到：

- 所有新需求；
- 风险提醒；
- AI 方案结果；
- 待处理任务。

#### B. Vendor Group

每个供应商一个群，或者一个供应商一个私聊/频道。

收到：

- 该供应商产品被询价；
- 该供应商产品被加入项目；
- 客户需要相关品类；
- 订单状态；
- 售后问题。

#### C. Project Group

每个成交项目一个群。

成员：

- 管理员；
- 客户代表；
- 供应商；
- 本地服务商；
- 施工人员。

群内自动同步：

- 项目摘要；
- 产品清单；
- 施工计划；
- 交付节点；
- 问题列表；
- 工单状态。

适合官方产品。

## 23. 关键商业建议

### 23.1 你不是卖货平台

你的平台应该说：

> We provide solution-ready devices with local integration and lifecycle support.

而不是：

> We sell cheap devices.

### 23.2 不要让供应商直接抢走客户

前期询盘可以由平台中转，不要默认开放客户联系方式。

供应商只看到必要项目信息，平台控制报价和沟通节奏。

### 23.3 服务包要标准化

不要每单都临时谈。

至少做：

- Basic
- Standard 3-Year
- Lifecycle 8-Year
- Premium 10-Year
- Custom

### 23.4 AI 只能辅助，不能最终承诺

所有 AI 方案必须标记：

> Preliminary recommendation, subject to engineering review and site verification.

### 23.5 前期案例可以用“方案样板”

你没有真实案例时，可以做：

- Concept solution
- Reference architecture
- Example package

不要伪造真实客户案例。

## 24. 最终推荐定位

AinerWise 最适合做成：

> B2B Smart Building & Energy Solution Platform

不是：

> 普通智能家居商城。

你的核心护城河应该是：

1. 中国供应链；
2. 欧洲本地服务；
3. AI 方案分析；
4. 系统集成能力；
5. 长期维护与质保；
6. 多区域复制能力。

前期从塞尔维亚做小项目、小服务、小维护开始，是现实的。

后期扩展到波兰、新西兰、澳洲，可以继续沿用同一套平台，只是换区域、语言、供应商、服务商和税费规则。

## 25. 一句话执行版

先不要做“大型 B2B 电商”。

先做：

> 高端解决方案官网 + B2B 需求表单 + 产品库 + 供应商库 + 后台项目管理 + Telegram 通知 + AI 初步方案助手。

等真实项目跑起来，再逐步加：报价、服务包、订单、工单、质保、供应商门户和多区域扩展。
