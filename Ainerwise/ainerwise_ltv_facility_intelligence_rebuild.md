# AinerWise 网站整改方案：Facility Intelligence + LTV 饲料模式平台

> 版本：v0.4  
> 用途：直接交给 Codex / Claude / Cursor / Windsurf / Gemini，对当前 AinerWise 网站和后台进行整改。  
> 核心目标：不要把 AinerWise 做成一次性卖设备的网站，而要升级成“设施智能化 + 合规监测 + 长期服务费 + 校准耗材 + 年度维护合同”的 LTV 平台。  
> 最重要一句话：前台用 BuildingBrain 讲未来智能建筑故事，后台用 StorageGuard / AquaGuard / KitchenGuard / EnergyGuard / FactoryPulse 跑长期现金流。

---

# 0. 最高优先级原则

AinerWise 不应该是普通电商，也不应该只是 B2B 设备目录。

真正定位应该是：

**AI Facility Intelligence + Lead-to-Solution + Recurring Service Platform**

中文理解：

**面向建筑、能源、仓储、工业、水质、商用厨房和高价值资产的 AI 设施智能化、合规监测和长期运维平台。**

核心收入不只是设备差价，而是：

- 初装设备费；
- 方案设计费；
- 安装实施费；
- SaaS / 平台费；
- 合规报表费；
- 年度校准费；
- 探头 / 标签 / 电池 / 耗材更换；
- 报警值守费；
- AMC 年度维护合同；
- 点位扩容费；
- 算法订阅费；
- 远程运维费；
- 供应商返点。

所有开发 AI 必须牢记：**AinerWise 不是卖便宜传感器，而是卖“持续监测、合规留痕、异常报警、年度维护和长期服务”。**

# 3. 商业优先级

## 3.1 第一优先级：StorageGuard

原因：

- 最像“饲料模式”；
- 温湿度合规、货损、审计记录都是刚需；
- 客户容易理解价值；
- 后期有校准、报表、报警、维护、传感器电池等持续收入。

适合客户：

- 医药仓；
- 食品冷库；
- 餐饮中央厨房；
- 酒店冷库；
- 超市冷链；
- 实验室；
- 花卉 / 种子仓储。

第一版必须重点做 StorageGuard 的展示页、AI Assessment 流程和 Demo 项目。

---

## 3.2 第二优先级：KitchenGuard

原因：

- 和酒店、餐饮、BuildingBrain 客户高度重叠；
- 燃气、CO、漏水、火灾都是高风险；
- 有年度检测、探头校准、报警通知、维护证书的长期收入；
- 比 AquaGuard 更容易理解和交付。

适合客户：

- 酒店后厨；
- 连锁餐厅；
- 商场餐饮；
- 中央厨房；
- 食品加工厨房。

---

## 3.3 第三优先级：AquaGuard

原因：

- 合规属性强；
- 探头耗材属性强；
- 长期服务费潜力大；
- 但需要更多水质、环保、校准专业知识，最好找合作伙伴。

适合客户：

- 工厂排水；
- 食品加工厂；
- 制药厂；
- 酒店泳池 / 水系统；
- 印染厂；
- 商业综合体污水。

---

## 3.4 第四优先级：BuildingBrain

作用：

- 品牌门面；
- 高端故事；
- 展示 AI、KNX、HVAC、光伏、储能、CCTV、数字人、本地 AI；
- 适合高客单项目，但售前更复杂。

---

## 3.5 第五优先级：FactoryPulse

作用：

- 中后期大单；
- 工厂 OEE、能耗、预测维护；
- 赚钱上限高，但售前和交付复杂，不作为第一现金流。

# 20. 最重要的产品原则

## 20.1 面向客户

不要说：

```text
我们卖传感器。
```

要说：

```text
我们帮你建立持续监测、合规留痕、异常报警和年度维护体系。
```

## 20.2 面向代码

不要把产品线建成普通商品分类。

要建成：

```text
Solution Line -> Monitoring Points -> Product Capability -> Proposal -> AMC -> Recurring Revenue
```

## 20.3 面向商业

AinerWise 真正赚钱的不是：

```text
一次性设备差价
```

而是：

```text
设备差价 + 平台费 + 校准费 + 耗材费 + 报表费 + 报警费 + 维护合同 + 扩容费
```

# 21. 最终结论

新版 AinerWise 应该是：

> 前台：未来 AI Facility Intelligence 品牌展示  
> 中台：AI 需求收集与方案评估  
> 后台：产品能力库、报价、Lead LTV、AMC、耗材与校准管理  
> 商业模式：一次性安装 + 长期合规 / 维护 / 耗材 / SaaS 收费

最适合的早期落地顺序：

1. StorageGuard：最像饲料模式，优先跑现金流；
2. KitchenGuard：酒店 / 餐饮 / 后厨，和 BuildingBrain 客户重合；
3. AquaGuard：合规强、耗材强，但需要更多专业合作；
4. BuildingBrain：品牌门面和高端项目；
5. EnergyGuard：和楼宇 / 工厂结合；
6. FactoryPulse：后期大单；
7. AssetPulse：有工地 / 医院 / 仓库资源时切入；
8. AgriBrain：未来扩展。
