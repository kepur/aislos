# AinerWise 当前系统改造 Skill：AI Smart Building Brain + Lead-to-Solution 平台

> 文件用途：请把本 Markdown 直接交给 Codex / Claude / Cursor / Windsurf / Gemini / 其他 AI 编程工具。  
> 目标：让其他 AI 在不了解完整历史对话的情况下，也能理解项目背景、用户职业规划、业务逻辑、当前系统问题、改造方向、页面设计、数据模型、开发优先级和验收标准。

# 0. 最高优先级原则

## 0.1 项目不是普通商城

AinerWise 不是 Amazon、Alibaba、普通 B2B 采购平台，也不是单纯产品目录。

它应该是：

> AI Smart Building Brain + Lead-to-Solution Platform

中文理解：

> 面向未来智能建筑的 AI 大脑展示站 + 客户需求收集 + AI 初步方案评估 + 产品能力库匹配 + 三档预算 + 付费方案转化 + 后期本地交付与长期维护平台。

用户最终卖的不是“便宜设备”，而是：

1. 方案设计；
2. 设备组合；
3. 本地实施；
4. 长期维护；
5. 中国供应链优势；
6. KNX / Home Assistant / Modbus / BACnet / MQTT / Matter 等系统集成能力；
7. AI 方案评估与未来 AI 楼宇大脑升级路线。

---

## 0.2 当前阶段不要做成完整 Marketplace

第一版不要做太重，不要立刻实现复杂撮合、支付托管、卖家竞价、完整交易闭环。

第一阶段重点是：

1. 炫酷前台展示；
2. AI Building Brain Demo；
3. Demo Buyer 登录；
4. AI Project Forge 智能建筑需求收集；
5. 产品能力库；
6. 高 / 中 / 低 / 未来定制四档方案；
7. 类 Excel 的 BOM / Budget Builder；
8. Lead Score；
9. Telegram 通知；
10. Phase-1 付费人工深化方案转化。

---

## 0.3 所有 AI 报价必须标注 Estimate

系统可以生成初步预算，但必须明确：

> AI estimate only. Final quote requires manual review, customer meeting, site survey, supplier confirmation, and signed contract.

中文：

> AI 预算仅为初步估算，最终报价必须经过人工审核、客户沟通、现场勘查、供应商确认和正式合同。

不得让客户误以为 AI 给出的价格就是最终合同价。

# 1. 用户背景与职业规划

开发 AI 必须理解用户的个人背景，否则容易把系统做偏。

## 1.1 用户背景

用户是网络与系统工程背景，长期关注：

- 网络工程；
- Linux / Docker / Kubernetes；
- 系统运维；
- AI 自动化；
- LangGraph / AI Agent 工作流；
- 本地模型部署；
- 系统集成；
- 中国供应链；
- 智能建筑；
- KNX；
- Home Assistant；
- 光伏 / 储能 / 能源管理；
- 东欧发展机会。

用户不是传统电工，也不是单纯销售。用户未来定位应该是：

> 懂 IT / AI / 网络 / 自动化 / 中国供应链的智能建筑系统集成商。

---

## 1.2 职业规划

用户计划路线大致是：

1. 当前阶段：准备 AinerWise 网站与系统，学习 KNX / ETS6 / 智能建筑知识；
2. 第一站：塞尔维亚低成本落地、考 KNX Basic、接触弱电 / 智能化 / CCTV / 网络 / 光伏监控 / HVAC 相关项目；
3. 建立本地关系：电工、KNX 小公司、HVAC 团队、装修团队、酒店 / 公寓 / 小商业客户、中国供应商；
4. 用小公司或个体户形式尝试承接项目；
5. 后续升级：波兰 / 东欧 / 欧盟市场，积累欧盟智能建筑经验；
6. 长期可能扩展：西班牙 / 葡萄牙 / 新西兰 / 澳洲等地区；
7. 商业主线：AinerWise 作为全球多区域智能建筑方案平台，先从 Serbia 起步，未来扩展到 Poland、New Zealand、Australia 等地区。

---

> 通过 IT + AI + KNX + 中国供应链 + 本地服务伙伴，成为轻量型智能建筑 / 能源 / 自动化系统集成商。

# 7. 智能等级体系 L1-L6

系统必须引入 Intelligence Level。

| Level | Name | 中文 | 能力 |
|---|---|---|---|
| L1 | Connected Building | 联网建筑 | 手机控制、基础远程控制、基础设备联网 |
| L2 | Sensor Automation | 传感器自动化 | 人体、门窗、温湿度、光照等联动 |
| L3 | Energy Optimized Building | 能源优化建筑 | 能耗监控、场景控制、节能策略、报表 |
| L4 | AI Assisted Building | AI 辅助建筑 | AI 分析、异常提醒、自动建议、数字人助手 |
| L5 | Local AI Building Brain | 本地 AI 建筑大脑 | 本地 AI、声纹 / 人脸权限、跨系统调度 |
| L6 | Autonomous Facility System | 半自主设施系统 | 机器人巡检、预测维护、自主能源优化 |

注意：

- L1-L3 是较容易落地的主力方案；

7. 不要让 AI 报价变成最终报价；
8. 不要让 UI 变得像廉价硬件商城；
9. 不要忽略长期服务费和备件；
10. 不要忽略用户的职业路线：塞尔维亚练手、波兰升级、长期做智能建筑系统集成。
